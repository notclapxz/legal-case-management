## FEATURE 2: Fix de Carpetas Jerárquicas

### 🎯 Problema

Los casos aparecían en la carpeta padre cuando en realidad solo debían mostrarse en la subcarpeta donde estaban asignados. Ejemplo:

**Antes** (❌ Incorrecto):
```
📁 Casos Laborales (muestra 10 casos)
  ├─ 📁 Despidos (4 casos)
  ├─ 📁 Accidentes (3 casos)
  └─ 📁 Demandas (3 casos)
```

**Ahora** (✅ Correcto):
```
📁 Casos Laborales (muestra 0 casos directos)
  ├─ 📁 Despidos (4 casos)
  ├─ 📁 Accidentes (3 casos)
  └─ 📁 Demandas (3 casos)
```

### ✅ Solución

Cambiar el filtrado de casos para mostrar **SOLO** los casos que tienen `carpeta_id` igual a la carpeta seleccionada, sin incluir subcarpetas recursivamente.

### 📝 Archivos Modificados/Creados

```
app/dashboard/casos/components/CasosConCarpetas.tsx   ✅ Refactorizado lógica de filtrado
app/dashboard/casos/components/CasosContenido.tsx     ✅ NUEVO componente
app/dashboard/casos/components/SidebarCarpetas.tsx    ✅ Carpetas expandidas por defecto
```

### 🔧 Cambio Clave

**Antes**:
```typescript
// ❌ Traía casos de carpeta + todas las subcarpetas
const idsIncluidos = getIdsRecursivos(carpetaSeleccionada)
const casosFiltrados = casos.filter(caso => 
  idsIncluidos.includes(caso.carpeta_id)
)
```

**Ahora**:
```typescript
// ✅ Solo casos DIRECTOS de la carpeta
const casosFiltrados = casos.filter(caso => 
  caso.carpeta_id === carpetaSeleccionada
)
```

### 🎨 Mejoras UI Agregadas

#### 1. **Breadcrumb de Navegación**
```

```
- Muestra la ruta completa de la carpeta actual

- Se actualiza automáticamente

#### 2. **Grid de Subcarpetas**

#### 3. **Carpetas Expandidas por Defecto**

## FEATURE 3: Auto-colapso del Sidebar en Casos

### 🎯 Objetivo

Maximizar el espacio horizontal en la vista de Casos ocultando automáticamente el sidebar principal (Dashboard, Casos, Agenda) pero manteniendo visible el sidebar de carpetas.

### ✅ Comportamiento Implementado

1. **Al entrar a `/dashboard/casos`**: Sidebar principal se oculta automáticamente
2. **Botón de toggle visible**: Flecha `>` en el lado izquierdo
3. **Usuario puede expandir**: Click expande el sidebar
4. **Estado manual persiste**: Si el usuario expande, queda expandido
5. **Al salir de Casos**: Vuelve al comportamiento normal

### 📝 Archivos Modificados

```
app/dashboard/components/DashboardLayoutWrapper.tsx  ✅ Lógica de auto-colapso
app/dashboard/components/Sidebar.tsx                 ✅ Props actualizados
app/dashboard/components/ToggleSidebarButton.tsx     ✅ Fixed warnings
```

### 🔧 Implementación Técnica

#### DashboardLayoutWrapper.tsx

```typescript
const [isCollapsed, setIsCollapsed] = useState(false)
const [manuallyExpanded, setManuallyExpanded] = useState(false)

// Detectar vista de casos
const isCasosView = pathname === '/dashboard/casos'

// Auto-colapsar si es vista de casos Y usuario no expandió manualmente
const shouldAutoCollapse = isCasosView && !manuallyExpanded
const effectiveCollapsed = shouldAutoCollapse || isCollapsed

// Toggle basado en estado efectivo
const toggleSidebar = () => {
  const newCollapsed = !effectiveCollapsed
  setIsCollapsed(newCollapsed)
  
  // Marcar como manualmente expandido si usuario expande en vista Casos
  if (isCasosView && !newCollapsed) {
    setManuallyExpanded(true)
  }
}

// Resetear al salir de la vista de casos
useEffect(() => {
  if (!isCasosView && manuallyExpanded) {
    setManuallyExpanded(false)
  }
}, [pathname])
```

### 🎨 Experiencia del Usuario

**Secuencia**:
1. Usuario hace click en "Casos" → Sidebar principal se oculta
2. Aparece botón con flecha `>` en el borde izquierdo
3. Usuario hace click en la flecha → Sidebar se expande
4. Usuario navega a "Agenda" → Sidebar vuelve a estado normal
5. Usuario vuelve a "Casos" → Sidebar se oculta de nuevo

## FEATURE 4: Sistema de Fotos de Perfil

### 🎯 Objetivo

Implementar un sistema completo para que cada usuario pueda subir y gestionar su foto de perfil, almacenada en Supabase Storage.

### 🗄️ Base de Datos

#### Migración Aplicada

**Archivo**: `add_avatar_url_to_profiles.sql`
```sql
ALTER TABLE profiles 
ADD COLUMN avatar_url TEXT;

COMMENT ON COLUMN profiles.avatar_url IS 
  'URL del avatar del usuario en Supabase Storage (bucket: avatars)';
```

#### Bucket de Storage Creado

```javascript
{
  id: 'avatars',
  name: 'avatars',
  public: true,              // Lectura pública
  file_size_limit: 2097152,  // 2 MB
  allowed_mime_types: [
    'image/jpeg',
    'image/png',
    'image/webp'
  ]
}
```

#### Estructura de Archivos

```
storage/avatars/
  ├── abc123-def456-ghi789.jpg    (UUID del usuario)
  ├── xyz789-abc123-def456.png
  └── ...
```

**Beneficio**: Un solo archivo por usuario, fácil de reemplazar y eliminar.

**Características**:

- ✅ Validación de formatos (JPG, PNG, WebP)
- ✅ Upload a Supabase Storage
- ✅ Update de BD (`profiles.avatar_url`)
- ✅ Elimina foto anterior automáticamente
- ✅ Botón "Eliminar Foto" (borra de Storage + BD)
- ✅ Estados de loading y error
- ✅ Optimizado con Next.js `<Image>`

**Flujo de Upload**:
```typescript
1. Usuario selecciona imagen
   ↓
2. Validar tamaño y formato
   ↓
3. Mostrar preview
   ↓
4. Usuario confirma → "Guardar Foto"
   ↓
5. Eliminar avatar anterior (si existe)
   ↓
6. Subir nuevo archivo a Storage
   ↓
7. Obtener URL pública
   ↓
8. Actualizar profiles.avatar_url
   ↓
9. Callback onUploadSuccess
   ↓
10. Cerrar modal + Refresh UI
```

**Código Clave**:
```typescript
// Upload a Supabase Storage
const fileExt = selectedFile.name.split('.').pop()
const fileName = `${user.id}.${fileExt}`

const { error: uploadError } = await supabase.storage
  .from('avatars')
  .upload(fileName, selectedFile, {
    cacheControl: '3600',
    upsert: true  // Sobrescribe si existe
  })

// Obtener URL pública
const { data: { publicUrl } } = supabase.storage
  .from('avatars')
  .getPublicUrl(fileName)

// Actualizar BD
await supabase
  .from('profiles')
  .update({ avatar_url: publicUrl })
  .eq('id', user.id)
```

### 🔒 Seguridad y Validaciones

#### Validaciones Frontend

```typescript
const MAX_FILE_SIZE = 2 * 1024 * 1024  // 2 MB
const ALLOWED_TYPES = ['image/jpeg', 'image/png', 'image/webp']

// Validar tipo
if (!ALLOWED_TYPES.includes(file.type)) {
  setError('Solo se permiten imágenes JPG, PNG o WebP')
  return
}

// Validar tamaño
if (file.size > MAX_FILE_SIZE) {
  setError('La imagen no puede superar 2 MB')
  return
}
```

#### Seguridad Backend (Supabase)

**Bucket Configuration**:
- ✅ `file_size_limit`: 2097152 bytes (2 MB)
- ✅ `allowed_mime_types`: Solo imágenes
- ✅ `public`: true (lectura pública, escritura protegida)

**RLS Policies** (conceptuales - Supabase maneja automáticamente):
- ✅ Lectura: Pública (todos pueden ver avatares)
- ✅ Escritura: Solo el dueño (`auth.uid() = user_id`)
- ✅ Eliminación: Solo el dueño
